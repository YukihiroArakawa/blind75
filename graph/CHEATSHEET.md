# blind75 Graph 領域 サマリー

各問題について「問題文サマリ」「着目すべき法則性 (Key Insight)」「解法サマリ」を整理する。
**計算量** は `時間 / 空間` の表記。`V`/`E` はノード数/辺数、`M×N` はグリッドサイズ。

## 133. Clone Graph

### 133.1. 問題文のサマリ

連結無向グラフのノード `node` を受け取り、**ディープコピー** したグラフを返す。各ノードは値と隣接ノードリストを持つ。

### 133.2. 着目すべき法則性

「同じ元ノードを 2 度クローンしない」「クローン同士の辺もコピーする」を両立するには、**`元ノード → クローンノード` の HashMap** を持って、訪問判定を兼ねさせるのが定石。

→ BFS/DFS で訪問しつつ、map にあれば再利用、なければ新規作成して再帰。

### 133.3. 解法のサマリ

- DFS 再帰: `clone(node)`
  - map にあればそれを返す
  - なければ新ノード作成 → map に登録 → `node.neighbors` をループしてクローンを `neighbors` に追加 (再帰)
- BFS でも同等に書ける
- **計算量**: O(V + E) / O(V)

## 207. Course Schedule

### 207.1. 問題文のサマリ

`numCourses` 個の科目と前提関係 `prerequisites[i] = [a, b]` (= a を取るには b が必要) が与えられたとき、全科目を履修可能か (true/false)。

### 207.2. 着目すべき法則性

依存関係を **有向グラフ** とみなし、**閉路がないこと (DAG)** が履修可能の必要十分条件。
判定方法は 2 通り:

- **Kahn の BFS**: 入次数 0 のノードから順に処理、最後に全ノード処理済なら DAG
- **DFS で循環検出**: 訪問中 (gray) を再訪したら閉路あり

### 207.3. 解法のサマリ

- Kahn 版:
  1. 隣接リスト + 入次数配列を構築
  2. 入次数 0 を Queue に詰める
  3. Queue から取り出す → 隣接ノードの入次数を -1、0 になったら enqueue
  4. 処理ノード数 == numCourses なら true
- **計算量**: O(V + E) / O(V + E)

## 417. Pacific Atlantic Water Flow

### 417.1. 問題文のサマリ

`m × n` の高さグリッド。**太平洋** (上端 + 左端) と **大西洋** (下端 + 右端) の両方に到達可能なセルを返す。
水は **同じか低い** セルにしか流れない。

### 417.2. 着目すべき法則性

順方向 (各セル → 海) で考えると全セルから探索が必要 → O((MN)²)。
**逆向き** (海 → 内陸へ「同じか高い方向」に流す) を 2 海それぞれで BFS/DFS すると O(MN) × 2 で済む。

→ 「太平洋から到達可能な集合」と「大西洋から到達可能な集合」の **積集合** が答え。

### 417.3. 解法のサマリ

- 2 つの真偽配列 `pacific[m][n]`, `atlantic[m][n]` を用意
- 上端・左端の各セルから DFS/BFS (高さ ≥ 元セル の方向に進む) で `pacific` を埋める
- 下端・右端から同様に `atlantic` を埋める
- 両方 true のセルが答え
- **計算量**: O(M × N) / O(M × N)

## 200. Number of Islands

### 200.1. 問題文のサマリ

`'1'` (陸) と `'0'` (海) からなるグリッドで、上下左右に隣接する陸の塊 (= 島) の数を返す。

### 200.2. 着目すべき法則性

陸セルから BFS/DFS で **連結成分** を 1 つ塗り潰す = 1 島。
全セルを走査し、未訪問の陸セルに出会うたびに「+1 してその島を全て訪問済みマーク」する。

→ 訪問マーキングは別配列 or **元グリッドの `'1'` を `'0'` に書き換え** でも OK (破壊的だが省メモリ)。

### 200.3. 解法のサマリ

- 二重ループで全セルを見る → `'1'` ならカウント++ して BFS/DFS で連結セル全てを `'0'` 化
- **計算量**: O(M × N) / O(M × N) (最悪、再帰スタック)

## 128. Longest Consecutive Sequence

### 128.1. 問題文のサマリ

未ソート整数配列の中で、**連続する整数列** (例: `[1, 2, 3, 4]`) の最長長さを返す。O(N) で。

### 128.2. 着目すべき法則性

各値を起点にナイーブに「+1 が存在するか」を伸ばすと O(N²)。
**「シーケンスの起点 (= `x - 1` が集合にない要素 `x`)」だけ伸長を試す** と、各シーケンスが 1 回しか走査されず合計 O(N)。

→ HashSet に全要素を入れた上で、`x - 1` が無い `x` を見つけたら `x, x+1, x+2, ...` を伸ばす。

### 128.3. 解法のサマリ

- HashSet 化
- `for x in nums: if (x-1) not in set: 伸ばして長さ計測 → best 更新`
- **計算量**: O(N) / O(N)

## 261. Graph Valid Tree

### 261.1. 問題文のサマリ

`n` ノードと無向辺リスト `edges` が与えられたとき、それが **木** (= 連結 + 閉路なし) か判定。

### 261.2. 着目すべき法則性

木の判定条件は次の 2 つで尽きる:

1. **辺数 = n - 1** (これより多いと閉路、少ないと非連結)
2. **連結である**

→ 辺数チェック 1 行 + 連結性チェック (BFS/DFS or Union-Find) だけ。
辺数 = n - 1 を満たせば「閉路あり ⇔ 非連結」が同値になるので、連結性確認だけで両方判定できる。

### 261.3. 解法のサマリ

- `edges.length != n - 1` なら即 false
- 隣接リスト構築 → ノード 0 から BFS/DFS → 訪問数 == n なら true
- 代替: Union-Find で各辺を union、既に同じ集合なら閉路 → false、最後に 1 集合なら true
- **計算量**: O(V + E) / O(V + E)

## 323. Number of Connected Components in an Undirected Graph

### 323.1. 問題文のサマリ

`n` ノードと無向辺リスト `edges` が与えられたとき、連結成分の数を返す。

### 323.2. 着目すべき法則性

**Union-Find** が最も直感的。各辺で `union(u, v)`、最後に「親が自分自身のノード数」が連結成分数。
代替: BFS/DFS で全ノードを巡回、「未訪問ノードから探索開始した回数」が連結成分数。

### 323.3. 解法のサマリ

- Union-Find: parent 配列を `[0..n-1]` で初期化、各辺で union → 親の異なる数を数える
- BFS/DFS: 未訪問のノードを見つけるたびにカウント++ して連結成分を全マーク
- **計算量**: O(V + E × α(V)) (Union-Find) / O(V)

## 269. Alien Dictionary

### 269.1. 問題文のサマリ

未知のアルファベットで書かれた辞書順ソート済みの単語リストが与えられたとき、文字の順序を 1 つ返す (順序が定まらないなら "")。

### 269.2. 着目すべき法則性

**隣接する 2 単語を比較**すると、**最初に異なる文字 1 ペア** から「先の方が辞書順で前」という辺が 1 つ得られる。

→ こうして得た文字グラフで **トポロジカルソート**:

- 矛盾 (循環) があれば不可能 → ""
- そうでなければトポソート結果が答え

エッジケース: `["abc", "ab"]` のように「長い方が前にある接頭辞関係」は不正 → ""。

### 269.3. 解法のサマリ

- 辺収集: 隣接単語ペアを順に比較し、最初の不一致文字で `(prev, curr)` の辺を追加
- Kahn のトポソート: 入次数 0 を Queue に → 順に取り出して結果に追加
- 最終長さ != 全文字数なら循環あり → ""
- **計算量**: O(C) (C = 全単語の総文字数) / O(1) (アルファベットサイズに比例)
