# 141. 連結リストの循環

難易度: Easy

## 問題

連結リストの先頭 `head` が与えられたとき、その連結リストに循環があるかどうかを判定してください。

連結リストにおいて、`next` ポインタを辿り続けることで再び到達できるノードが存在する場合、その連結リストは循環を持ちます。内部的には、末尾ノードの `next` ポインタが接続されているノードのインデックスを `pos` で表します。**ただし、`pos` は引数として渡されません。**

連結リストに循環がある場合は `true`、そうでない場合は `false` を返してください。

## 例

**例 1:**

```
入力: head = [3,2,0,-4], pos = 1
出力: true
説明: 連結リストに循環があり、末尾ノードが 1 番目のノード（0-indexed）に接続されています。
```

**例 2:**

```
入力: head = [1,2], pos = 0
出力: true
説明: 連結リストに循環があり、末尾ノードが 0 番目のノードに接続されています。
```

**例 3:**

```
入力: head = [1], pos = -1
出力: false
説明: 連結リストに循環はありません。
```

## 制約

- リスト内のノード数の範囲は `[0, 10^4]`
- `-10^5 <= Node.val <= 10^5`
- `pos` は `-1` または連結リスト内の**有効なインデックス**
