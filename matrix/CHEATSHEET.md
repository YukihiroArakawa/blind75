# blind75 Matrix 領域 サマリー

各問題について「問題文サマリ」「着目すべき法則性 (Key Insight)」「解法サマリ」を整理する。
**計算量** は `時間 / 空間` の表記。`M × N` はグリッドサイズ。

行列問題の典型パターン:
- **In-place 変換**: 追加メモリを使わず元配列の一部をフラグストレージに使う
- **境界縮小 / 4 方向制御**: spiral, rotation などで境界を変数で管理
- **DFS + Backtracking**: グリッド上の経路探索

## 73. Set Matrix Zeroes

### 73.1. 問題文のサマリ

`m × n` 行列で、要素が 0 のセルがあれば、その **行全体と列全体を 0** にする (in-place)。

### 73.2. 着目すべき法則性

素朴に「0 のある行/列を別配列で記録 → 後で 0 化」だと O(M+N) 空間。
**1 行目と 1 列目をフラグストレージに転用** すると追加 O(1) 空間で済む:

- 行 i の 1 列目セル `matrix[i][0]` が 0 → 行 i を 0 化すべき
- 列 j の 1 行目セル `matrix[0][j]` が 0 → 列 j を 0 化すべき
- ただし「1 行目自体が元から 0 を含むか」「1 列目自体が元から 0 を含むか」を別変数で先に記録する必要あり (フラグ書き込みで上書きされるため)

### 73.3. 解法のサマリ

1. `firstRowZero`, `firstColZero` を別 bool で記録
2. 内側 (`i ≥ 1, j ≥ 1`) を走査 → 0 を見つけたら `matrix[i][0] = 0; matrix[0][j] = 0`
3. 内側を再走査 → `matrix[i][0] == 0 || matrix[0][j] == 0` なら 0 化
4. `firstRowZero` に応じて 1 行目を、`firstColZero` に応じて 1 列目を 0 化
- **計算量**: O(M × N) / O(1)

## 54. Spiral Matrix

### 54.1. 問題文のサマリ

`m × n` 行列を **外側から渦巻き状** (右 → 下 → 左 → 上 → 右 → ...) に走査した値リストを返す。

### 54.2. 着目すべき法則性

**4 つの境界 `top, bottom, left, right`** を変数で管理し、1 周ごとに内側に縮める:

1. `top` 行を `left → right` で走査 → `top++`
2. `right` 列を `top → bottom` で走査 → `right--`
3. `bottom` 行を `right → left` で走査 (条件: `top <= bottom`) → `bottom--`
4. `left` 列を `bottom → top` で走査 (条件: `left <= right`) → `left++`

3, 4 で条件チェックが必要なのは、矩形の最後に「1 行のみ」または「1 列のみ」になったときの二重走査防止。

### 54.3. 解法のサマリ

- 4 境界変数を初期化、`top <= bottom && left <= right` の間 4 ステップを順に実行
- **計算量**: O(M × N) / O(1) (出力除く)

## 48. Rotate Image

### 48.1. 問題文のサマリ

`n × n` 行列を **時計回り 90 度回転** する (in-place、追加メモリ最小)。

### 48.2. 着目すべき法則性

90 度回転 = **転置 → 各行を反転** という 2 操作の合成で実現できる:

```
[1 2 3]    転置    [1 4 7]    各行反転    [7 4 1]
[4 5 6]   ----→   [2 5 8]   --------→  [8 5 2]
[7 8 9]            [3 6 9]               [9 6 3]
```

転置は対角線上で対称要素 swap、行反転は両端から swap。どちらも in-place で O(N²)。

### 48.3. 解法のサマリ

- 転置: `for i = 0..n-1, for j = i+1..n-1: swap(matrix[i][j], matrix[j][i])`
- 各行反転: `for row in matrix: row を両端から swap`
- 反時計回り 90° なら「転置 → 各 **列** 反転」または「各行反転 → 転置」
- **計算量**: O(N²) / O(1)

## 79. Word Search

### 79.1. 問題文のサマリ

文字グリッドと単語 `word` が与えられたとき、上下左右に隣接 (同セル再使用不可) で `word` を綴れるパスが存在するか判定。

### 79.2. 着目すべき法則性

**各セルを起点に DFS + Backtracking** で `word` の各文字を順に消費していく。
訪問済み管理は **元グリッドの文字を一時的にダミー文字 (例 `'#'`) に書き換え → 戻すときに復元** で追加メモリ不要。

枝刈り: 現在の文字が word の対応位置と異なれば即 return。

### 79.3. 解法のサマリ

- 二重ループで全セルを起点に DFS 試行
- DFS `(r, c, k)`:
  - `k == word.length` なら true
  - 範囲外 / 文字不一致 なら false
  - `original = grid[r][c]; grid[r][c] = '#'`
  - 4 方向再帰、1 つでも true なら true
  - `grid[r][c] = original` (復元)
- **計算量**: O(M × N × 4^L) (L = word 長) / O(L) (再帰深さ)
