# 33. 回転ソート済み配列の探索

難易度: Medium

## 問題

昇順にソートされた整数配列 `nums` があります（値はすべて**異なる**）。

関数に渡される前に、`nums` は未知のインデックス `k`（`1 <= k < nums.length`）で**左回転されている可能性があります**。その結果、配列は `[nums[k], nums[k+1], ..., nums[n-1], nums[0], nums[1], ..., nums[k-1]]`（**0-indexed**）となります。例えば、`[0,1,2,4,5,6,7]` を `3` インデックス分左回転すると `[4,5,6,7,0,1,2]` になります。

回転後の配列 `nums` と整数 `target` が与えられたとき、`target` が `nums` に含まれていればそのインデックスを返し、含まれていなければ `-1` を返してください。

`O(log n)` の実行時計算量を持つアルゴリズムで記述する必要があります。

## 例

**例 1:**

```
入力: nums = [4,5,6,7,0,1,2], target = 0
出力: 4
```

**例 2:**

```
入力: nums = [4,5,6,7,0,1,2], target = 3
出力: -1
```

**例 3:**

```
入力: nums = [1], target = 0
出力: -1
```

## 制約

- `1 <= nums.length <= 5000`
- `-10^4 <= nums[i] <= 10^4`
- `nums` のすべての値は**ユニーク**です。
- `nums` は回転されている可能性のある昇順配列です。
- `-10^4 <= target <= 10^4`
