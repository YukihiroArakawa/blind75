# blind75 Binary 領域 サマリー

各問題について「問題文サマリ」「着目すべき法則性 (Key Insight)」「解法サマリ」を整理する。
**計算量** は `時間 / 空間` の表記。bit 操作問題は時間 O(1) (固定 32 bit) のものが多い。

## 索引

| # | 問題 | 難易度 | 中心パターン |

## 191. Number of 1 Bits

### 191.1. 問題文のサマリ

符号なし 32bit 整数 `n` のビット表現に含まれる **1 の個数** (popcount, ハミング重み) を返す。

### 191.2. 着目すべき法則性

**`n & (n - 1)` は `n` の最下位の 1 ビットを 1 つだけ消す**。
→ 0 になるまで繰り返した回数 = 1 の個数。
全 32 bit を順に見るより、1 ビット消すごとにループ 1 回で済むので 1 の個数分だけしか反復しない。

### 191.3. 解法のサマリ

- ループ: `count++; n = n & (n - 1);` を `n != 0` の間繰り返す
- **計算量**: O(k) / O(1) (k = 1 ビットの個数、最大 32)

## 338. Counting Bits

### 338.1. 問題文のサマリ

整数 `n` に対し、`0 ≤ i ≤ n` の各 `i` について popcount を計算した配列を返す。
O(N) 時間で。

### 338.2. 着目すべき法則性

`i` の popcount は **`i >> 1` の popcount** と最下位ビットの和:

```
bits[i] = bits[i >> 1] + (i & 1)
```

理由: `i` を 1 ビット右シフトすると最下位が落ちる → 残りの popcount に最下位を足し直せば良い。
DP テーブルに過去の結果を持てば各 `i` の計算が O(1)。

### 338.3. 解法のサマリ

- 配列 `bits[0..n]` を 0 で初期化、`i = 1..n` で `bits[i] = bits[i >> 1] + (i & 1)`
- **計算量**: O(N) / O(N)

## 268. Missing Number

### 268.1. 問題文のサマリ

`[0, n]` のうち `n` 個の異なる整数を含む配列 `nums` (= 1 つだけ欠けている) から、欠けた数を返す。

### 268.2. 着目すべき法則性

**XOR の自己消去性** (`a ^ a = 0`, `a ^ 0 = a`) を活用:

- `0 ^ 1 ^ 2 ^ ... ^ n` と `nums の全要素` を XOR すると、ペアになるものは消えて欠けた数だけ残る

代替案: 期待和 `n*(n+1)/2` から実際の和を引く。XOR の方がオーバーフロー耐性あり。

### 268.3. 解法のサマリ

- XOR 法: `result = n` で初期化、`result ^= i ^ nums[i]` を全 `i` で
- **計算量**: O(N) / O(1)

## 190. Reverse Bits

### 190.1. 問題文のサマリ

符号なし 32bit 整数のビット並びを反転 (MSB と LSB を入れ替え) した値を返す。

### 190.2. 着目すべき法則性

「LSB を取り出す → 結果に MSB として置く」を 32 回繰り返す。
具体的には 1 ステップで:

1. `result = result << 1` (結果を左に空ける)
2. `result = result | (n & 1)` (n の LSB を結果の LSB に置く)
3. `n = n >> 1` (n を右シフト)

これだけで反転完成。ビットごとに 1 対 1 で対応する。

### 190.3. 解法のサマリ

- 32 回ループで上記 3 ステップ
- 高速化: 8 ビット単位の lookup テーブル (面接では暗記不要)
- **計算量**: O(1) (固定 32 ループ) / O(1)

---

## 371. Sum of Two Integers

### 371.1. 問題文のサマリ

整数 `a`, `b` の和を、**`+`, `-` 演算子を使わず** に返す。

### 371.2. 着目すべき法則性

加算をビット演算に分解:

- **桁ごとの和 (繰上り無視)** = `a XOR b`
- **繰上り** = `(a AND b) << 1`

→ 「桁の和」と「繰上り」を再度足す → 繰上りが 0 になるまで繰り返せば完成 (筆算と同じ)。
Java では int 32 bit で自然にオーバーフロー処理されるので符号付きでも動く。

### 371.3. 解法のサマリ

- ループ: `b != 0` の間
  - `carry = (a & b) << 1`
  - `a = a ^ b`
  - `b = carry`
- 終了後 `a` が答え
- **計算量**: O(1) (繰上りは最大 32 回伝播) / O(1)
